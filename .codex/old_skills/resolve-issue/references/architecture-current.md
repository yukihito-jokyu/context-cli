# Current Architecture Update

## Responsibility

- 実装済みシステムの現在状態だけを記述する。
- 将来案、未実装設計、Issue上の希望を含めない。

## Sources

- 実装コード、設定、CI、Accepted済みADR
- 実装済みArchitecture Change

## Update Rules

- Issueの結論が実装済みであることを確認できる場合だけ反映する。
- 未実装ならArchitecture ChangeまたはTasksへ反映し、現在Architectureは変更しない。
- 事実同期では`Accepted`を維持できるが、根拠となるコードを必ず確認する。

## Verification

- 記述がコード、設定、実行経路と一致する。
- 将来形と現在形を混在させていない。
- 標準Architecture文書間の責務と参照が整合する。
