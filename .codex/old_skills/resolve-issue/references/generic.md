# Generic File Update

## Responsibility

- ファイルの既存目的、テンプレート、所有者を特定し、その責務内だけを更新する。

## Sources

- ファイル自身
- 直接参照する正本資料
- 関連するコード、設定、CI

## Update Rules

- Statusやテンプレートを独自追加しない。
- Issueの結論を最小差分で反映する。
- 責務が判定できない場合は修正せず、人間へ確認する。
- 主要成果物をgeneric規則だけでAccepted化しない。

## Verification

- 既存形式と責務を維持する。
- 入力外ファイルや無関係な内容を変更していない。
