# PRD Update

## Responsibility

- 1つのユーザー成果についてGoal、Problem、Scope、成功判定、前提を保持する。
- Story分解、技術要件、実装方式を含めない。

## Sources

- `docs/ai-driven-development/templates/prd.md`
- `docs/product.md`
- 関連Issueで確定したユーザー価値とScope

## Update Rules

- Goal、Problem、Target User、Success Metrics、Scope、Out of Scopeへの影響を明示する。
- 意味を変更した場合は`Draft`へ戻し、Backlog以降の再確認対象を列挙する。
- Issue解決が既存Scope内の詳細確定だけならPRDを変更しない。

## Verification

- 1つの独立したユーザー成果として成功判定できる。
- Solutionや実装詳細が混入していない。
- Productと矛盾せず、下流のStoryへ追跡できる。
