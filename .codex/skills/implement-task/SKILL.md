# タスク実装

設計承認済みのタスクを実装し、レビュー可能な状態にする。

## 入力

- 実装対象のタスクファイルパス（例: `docs/specs/spec-XXX-<slug>/tasks/01-init.md`）

## 1. 準備

- 対象タスク、`spec.md`、[AGENTS.md](../../../AGENTS.md)、関連コードとテストを読む。
- ステータスが `仕掛中 (In Progress)` であることを確認する。状態遷移は親エージェントに任せる。

## 2. サイクル実行

1. 完成条件と失敗経路を検証するテストを先に追加する。
2. テストを通す最小限の内容を実装する。
3. 変更範囲を整理し、回帰テストを実行する。
4. [AGENTS.md](../../../AGENTS.md) の品質ゲートをすべて実行する。

## 3. 進捗更新

- 実装チェックリストを随時更新する。
- 実際の変更ファイルと検証結果をタスクファイルへ反映する。

## 4. 完了

- 変更ファイル、実装内容、実行した品質ゲートと結果を親エージェントへ報告する。
- ステータスを直接変更しない。
