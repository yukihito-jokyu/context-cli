# 仕様書準備フロー実行 (Prepare Spec Flow)

新規開発の開始時に、仕様策定（spec.md）からタスクの切り出しまでを最小限の手順で実行・管理する。

## 入力

- 解決したいユーザー課題・要件（新規の場合）
- もしくは再開対象の仕様書ID（再開の場合）

## フロー

1. [define-spec](../define-spec/SKILL.md) で `spec.md` を作成または更新する。
2. `spec_reviewer` に [review-spec](../review-spec/SKILL.md) を実行させる。
3. 指摘があれば親エージェントが改善案をユーザーと合意し、仕様書を修正して再レビューする。
4. 指摘解消後、ユーザーから仕様承認を得る。
5. [plan-tasks](../plan-tasks/SKILL.md) でタスクを生成する。
6. タスク一覧と次に開始するタスクを報告する。

レビューサブエージェントはファイルを変更しない。承認前にタスク分割へ進まない。
