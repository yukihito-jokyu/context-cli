# 仕様書・設計準備フロー実行 (Prepare Spec Flow)

新規開発の開始時に、仕様策定（spec.md）からタスクの切り出し・具体設計およびレビューまでを最小限の手順で実行・管理する。

## 入力

- 解決したいユーザー課題・要件（新規の場合）
- もしくは再開対象の仕様書ID（再開の場合）

## フロー

1. [define-spec](../define-spec/SKILL.md) で `spec.md` を作成または更新する。
2. `spec_reviewer` に [review-spec](../review-spec/SKILL.md) を実行させる。
3. 指摘があれば親エージェントが改善案をユーザーと合意し、仕様書を修正して再レビューする。
4. 指摘解消後、ユーザーから仕様承認を得る。
5. [design-tasks](../design-tasks/SKILL.md) でタスク分解と各タスクの具体設計を一括で生成する。
6. `task_design_reviewer` に [review-tasks-design](../review-tasks-design/SKILL.md) を実行させる。
7. 指摘があれば親エージェントが改善案をユーザーと合意し、設計を修正して再レビューする。
8. 設計レビュー合格後、ユーザーから設計承認（タスク分解および各タスクの具体設計の承認）を得る。
9. 承認されたタスク一覧と、最初に開始するタスクを報告する。

レビューサブエージェントはファイルを変更しない。承認前に実装フェーズへ進まない。
