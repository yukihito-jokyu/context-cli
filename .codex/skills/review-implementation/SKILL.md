# 実装コードレビュー

実装された機能コードおよびテストコードの品質を担保し、不具合の流出を防止する。

## 入力

- 対象のタスクファイルパス（例: `docs/specs/spec-XXX-<slug>/tasks/01-init.md`）
- 変更されたソースコードおよびテストコードのGit差分

## 1. レビュー観点

次を確認する。

- `spec.md` とタスク設計の完成条件を満たしているか。
- 実装とテストが [AGENTS.md](../../../AGENTS.md) に適合するか。
- バグ、回帰、エラー処理漏れ、セキュリティ問題がないか。
- 正常系、異常系、境界条件のテストが十分か。

## 2. フィードバックと改善

- 指摘を重大度順に、ファイルと行、根拠、改善案付きで親エージェントへ報告する。
- コード、テスト、タスクファイル、ステータスを変更しない。
- 指摘がない場合も、残存リスクと未実行の検証を報告する。
