# タスク設計レビュー

個別タスク設計を第三者視点で検証し、実装へ進める品質かを判定する。

## 入力

- レビュー対象のタスクファイルパス（例: `docs/specs/spec-XXX-<slug>/tasks/01-init.md`）

## 1. レビュー観点

次を確認する。

- 対象ストーリーと完成条件を単一の縦スライスとして実装できるか。
- 正常系、異常系、境界条件がフローとシーケンスに含まれるか。
- ファイル配置と責務が [AGENTS.md](../../../AGENTS.md) に適合するか。
- テスト計画に具体的なシナリオ、対象、実行方法があるか。

## 2. フィードバックと改善

- 重大度、根拠、改善案を整理して親エージェントへ報告する。
- ファイルやステータスを変更せず、実装へ進めるかを明示する。
