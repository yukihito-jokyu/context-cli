# ドキュメント更新

実装結果を反映して設計・開発関連のドキュメント（`docs/`）を更新し、設計書とソースコードが完全に同期された状態を保つ。

## 入力

- 完了したPRD ID（および `prd.md` のパス）

## 1. 影響範囲を調査する

Gitの変更履歴およびコードを調査し、以下のドキュメントのうちどれに影響があるかを特定する。

- `docs/product.md`: プロダクトの概要、目的、ユースケース、要件定義
- `docs/engineering/structure.md`: ディレクトリ構造、コンポーネント配置、モジュール境界と依存ルール
- `docs/engineering/technology.md`: 採用している技術スタック、技術的制約
- `docs/engineering/development-rules.md`: 命名規則、実装ルール、エラー処理、セキュリティ、テスト規則
- `docs/engineering/bootstrap.md`: プロジェクト開発環境の初期構築ログ

## 2. 該当ファイルを更新する

影響を受けたファイルのみを更新する。

- 更新時は、変更箇所の内容をソースコードの具象的な実装（構造体名、パッケージ名、データフローなど）と完全に同期させる。
- **※注意**: 各ファイルヘッダーに `Status: Draft/Accepted` などのメタデータは勝手に変更しない。
- 影響のないファイルは更新せず、そのままとする。

## 3. 完了報告

- 更新したドキュメントファイルの一覧と、どのような変更内容を反映したかを人間に報告する。
